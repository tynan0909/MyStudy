캡슐화 상속 다형성


상속

public class A extends B 의 의미? -> B에서 확장된, 상속된 A(B의 DLC가 A)
원래는 남의 클래스를 이용하려면 new로 불러와야 했지만
이거로도 가능 -> 즉 new를 안쓰고 메모리에 올릴 수 있다.

즉 A가 작동하면 그전에 B가 자동으로 켜진다
=> public class 자식 extends 부모 extends 조부모

ex)public class 영등포점 extends 서울
public class 서울 extends 스벅코리아
public class 스벅코리아 extends 스벅본사
도 가능함

이렇게 클래스를 쪼개는 이유?
클래스가 비대해지는거 방지, 효율화

다형성을 쓰려면(남의스킬배껴쓰려면)
상속도 필요하다?

상속을 is a 관계라고도 함(스벅영등포점 is a 스벅)집합?
new는 has a관계라고 함(손님 has a coin)

private	-	내 클래스에서만 쓸거임
public	-	다른 패키지에서도 씀
default	-	같은 패키지에서 씀
protected	-	다른패키지에서도 씀

프로텍티드는 상속을 사용할걸 염두에 둔거임

클래스에 final 붙이면 다른데서 상속받을 수 없음(독신주의자임)
그래서 이경우엔 new를 써서 불러야함

this는 내(클래스내부)에서 나를 보는 생성자
super는 자식이 부모에게 상속받을 때 쓸수있음(상속에서 쓰는 this)
그리고 super는 맨위에 있어야함


접근 제어자		같은 클래스	같은 패키지	다른 패키지의 자식 클래스	다른 패키지의 일반 클래스	설명
private				✅		❌			❌					❌	클래스 내부에서만 접근 가능
default (package-private)	✅		✅			❌					❌	접근 제어자 생략 시, 같은 패키지에서만 접근 가능
protected				✅		✅			✅					❌	같은 패키지 + 다른 패키지의 자식 클래스에서 접근 가능
public				✅		✅			✅					✅	모든 클래스에서 접근 가능
